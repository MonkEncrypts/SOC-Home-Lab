# Investigation Outcome

The investigation successfully reconstructed the simulated reconnaissance activity using endpoint telemetry collected by Sysmon and analyzed within Splunk Enterprise.

Evidence confirmed that the executed Windows command-line utilities generated Process Create events, allowing the complete execution sequence to be observed and correlated throughout the investigation.

Although each executed command represents legitimate Windows functionality, their combined execution pattern aligns with reconnaissance behavior commonly performed after an attacker gains interactive access to a system.

---

# Evidence-Based Observations

| Evidence | Observation | Analyst Interpretation |
|----------|-------------|-------------------------|
| `whoami.exe` execution | Current user was enumerated | Indicates the attacker is identifying the security context of the compromised session. |
| `hostname.exe` execution | Hostname retrieved | Suggests the attacker is identifying the compromised endpoint. |
| `ipconfig.exe` execution | Network configuration collected | Indicates network reconnaissance and environment discovery. |
| `systeminfo.exe` execution | System information gathered | Enables operating system and hardware profiling. |
| `net user` execution | Local user accounts enumerated | May support privilege escalation or lateral movement planning. |
| `tasklist.exe` execution | Running processes enumerated | Provides visibility into active applications and potential security controls. |

---

# Behavioral Assessment

No individual command observed during this investigation was inherently malicious.

Each executable (`whoami.exe`, `hostname.exe`, `ipconfig.exe`, `systeminfo.exe`, `net.exe`, and `tasklist.exe`) is a legitimate Windows utility commonly used by administrators, support engineers, and end users.

However, when evaluated collectively, the sequence of executed commands demonstrates a structured reconnaissance workflow that closely resembles attacker behavior following successful initial access.

Rather than evaluating each event independently, the investigation considered the complete execution sequence, parent-child process relationships, execution context, and overall attack timeline.

> [!IMPORTANT]
> Effective SOC investigations rely on behavioral analysis rather than signature-based thinking. Multiple legitimate administrative commands executed in rapid succession may collectively represent malicious reconnaissance, even though each command appears benign in isolation.

---

# SOC Analyst Perspective

One of the primary responsibilities of a SOC analyst is distinguishing legitimate administrative activity from adversary behavior.

This investigation demonstrates that process execution should always be interpreted within its surrounding context rather than evaluated as isolated events.

Factors such as execution sequence, user context, parent-child process relationships, frequency of execution, and accompanying telemetry provide significantly more investigative value than the executable name alone.

This contextual approach reduces false positives while improving the analyst's ability to identify genuine threats.

---

# Investigation Conclusion

The collected evidence supports the conclusion that the simulated attack successfully reproduced a realistic host reconnaissance scenario using legitimate Windows command-line utilities.

The telemetry generated by Sysmon and ingested into Splunk Enterprise provided sufficient visibility to reconstruct the attack sequence from endpoint execution through SIEM investigation.

This investigation reinforces the importance of contextual analysis when evaluating command-line activity and demonstrates how endpoint telemetry can be leveraged to identify behavioral patterns associated with early-stage attacker reconnaissance.
