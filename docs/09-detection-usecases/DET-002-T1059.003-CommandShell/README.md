# Executive Summary

This investigation demonstrates how native Windows Command Shell activity can be detected, validated, and investigated using endpoint telemetry generated by Sysmon and analyzed within Splunk Enterprise.

A realistic post-compromise reconnaissance scenario was simulated by executing legitimate Windows command-line utilities commonly abused by adversaries during the discovery phase of an intrusion.

Rather than focusing solely on process execution, this investigation emphasizes the complete SOC investigation lifecycle—from attack simulation through telemetry validation, SIEM investigation, behavioural analysis, and detection engineering.

The objective of this case study is to demonstrate practical SOC analyst skills while reinforcing the importance of contextual analysis when investigating command-line activity.

---

# Investigation Objectives

The investigation was designed to answer the following questions:

- Can Sysmon successfully capture Windows Command Shell activity?
- Is endpoint telemetry successfully forwarded to Splunk Enterprise?
- Can the complete attack sequence be reconstructed from collected telemetry?
- Which telemetry fields provide the greatest investigative value?
- How should a SOC analyst distinguish legitimate administrative activity from attacker reconnaissance?
