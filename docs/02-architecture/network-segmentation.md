# Network Segmentation Design Rationale

## Why Two Networks?

A SOC home lab is most useful when it behaves like a real environment: attack traffic should be contained, monitoring traffic should be reliable, and the lab should still be maintainable (patchable, updatable) without exposing simulated attacks to the open internet. A single flat network cannot satisfy both requirements at once, so this lab uses two purpose-built VMware virtual networks.

---

## VMnet1 — Host-Only Network (Internal SOC Network)

| Attribute | Value |
|---|---|
| Purpose | Internal SOC communication — attacker, endpoint, and SIEM |
| Network | 192.168.13.0/24 |
| Gateway / SIEM Host | 192.168.13.1 |
| Windows 10 Endpoint | 192.168.13.128 |
| Kali Linux Attacker | 192.168.13.129 |

VMware Host-Only networking creates a private virtual switch that connects VMs to each other and to the host, but **not** to the physical LAN or internet. This guarantees that any attack traffic generated by the Kali VM — port scans, exploitation attempts, lateral movement simulations — stays entirely inside the lab.

## VMnet8 — NAT Network (Internet Access)

| Attribute | Value |
|---|---|
| Purpose | Internet access for OS updates and tool installation |
| Network | 192.168.72.0/24 |
| Components | Windows 10 NAT adapter, Kali NAT adapter |

Each VM has a second virtual NIC bound to VMware's NAT network. This adapter is used exclusively for outbound internet access — downloading Sysmon, the Splunk Universal Forwarder installer, Kali tool updates, and Windows updates. It is never used to carry attack or telemetry traffic.

---

## Why This Matters for a SOC Analyst Role

In production environments, SOC analysts work with network segmentation constantly: VLANs separating user, server, and management traffic; jump boxes bridging isolated security tooling networks; and DMZs isolating internet-facing systems. Designing this lab with the same separation-of-concerns principle — isolating "dirty" attack traffic from "clean" internet/update traffic — demonstrates an understanding of network segmentation as a security control, not just as a lab convenience.

This also mirrors a real detection engineering consideration: if Sysmon or Splunk telemetry showed unexpected traffic on the NAT adapter that should only be active during a Windows Update check, that would itself be worth investigating as a potential indicator of compromise.
